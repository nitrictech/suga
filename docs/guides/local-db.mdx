---
title: "Local Database Setup"
description: "Set up PostgreSQL for Nitric local development."
---

Nitric doesn't automatically provision databases in local development - this gives you full control over your database choice. Here's how to set up PostgreSQL locally that works with any PostgreSQL provider (Neon, Supabase, AWS RDS) when deployed.

<AccordionGroup>
  <Accordion title="Step 1: Initialize Project" icon="folder-plus">
    **Start with a fresh project:**

    Create a new project and select the Go template:

    ```bash
    nitric new my-app
    ```

    When prompted, select the Go template:

    ```bash
    Welcome to Suga, this command will help you create a project from a template.
    
    Project name: my-app
    
    ? Template: 
      suga/python
    > suga/go          # Select this one
      suga/typescript
    
    ✓ Project created!
    ```

    Enter your project:

    ```bash
    cd my-app
    ```
  </Accordion>
  <Accordion title="Step 2: Setup Database Environment" icon="docker">
    **Run PostgreSQL locally without installing it:**

    <Tip>
      Already have PostgreSQL running? Skip the Docker setup and use your existing database. Just update the `DATABASE_URL` in Step 3 to point to your instance.
    </Tip>
    Create `docker-compose.yml` for PostgreSQL:

    ```yaml
    services:
      postgres:
        image: postgres:15
        environment:
          POSTGRES_USER: ${POSTGRES_USER:-admin}
          POSTGRES_PASSWORD: ${POSTGRES_PASSWORD:-secure_password}
          POSTGRES_DB: ${POSTGRES_DB:-myapp}
        ports:
          - '5432:5432'
        volumes:
          - postgres_data:/var/lib/postgresql/data
          - ./init.sql:/init.sql
    
    volumes:
      postgres_data:
    ```

    Create an `.env` file for local development (don't commit this):

    ```bash
    POSTGRES_USER=admin
    POSTGRES_PASSWORD=secure_password
    POSTGRES_DB=myapp
    ```

    Add database commands to your `Makefile`:

    ```makefile
    .PHONY: db-start db-stop
    
    db-start:
    	@echo "🐳 Starting database..."
    	@docker compose up -d
    	@echo "✅ Database ready!"
    
    db-stop:
    	@echo "🛑 Stopping database..."
    	@docker compose down
    ```
  </Accordion>
  <Accordion title="Step 3: Configure & Connect Nitric" icon="plug">
    **Tell Nitric how to connect to your database:**

    Add your database to `nitric.yaml`:

    ```yaml
    databases:
      mydb:
        env_var_key: DATABASE_URL
        access:
          app:
            - query
    
    services:
      app:
        env:
          DATABASE_URL: postgresql://admin:secure_password@localhost:5432/myapp?sslmode=disable
    ```

    <Note>
      Update the DATABASE_URL to match your .env file values. For production, use environment variables or secrets management.
    </Note>
    Start the database and launch Nitric:

    ```bash
    make db-start
    nitric dev
    ```

    Nitric will now connect to your PostgreSQL database automatically.
  </Accordion>
  <Accordion title="Step 4: Initialize Database" icon="table">
    **Create your tables with sample data:**

    Create `init.sql` with your schema and initial data:

    ```sql
    DROP TABLE IF EXISTS users;
    
    CREATE TABLE users (
      id SERIAL PRIMARY KEY,
      email VARCHAR(255) UNIQUE NOT NULL,
      created_at TIMESTAMP DEFAULT NOW()
    );
    
    INSERT INTO users (email) VALUES
      ('admin@example.com'),
      ('user@example.com');
    ```

    Add the reset command to your Makefile:

    ```makefile
    db-reset:
    	@echo "🔄 Resetting database..."
    	@docker compose down -v
    	@docker compose up -d
    	@sleep 2
    	@docker compose exec postgres psql -U admin -d myapp -f /init.sql
    	@echo "✅ Database reset with fresh data!"
    ```

    Apply your schema:

    ```bash
    make db-reset
    ```

    <Note>
      For production, use a proper migration tool like golang-migrate, Flyway, or your ORM's migration system.
    </Note>
  </Accordion>
  <Accordion title="Step 5: Query from API" icon="code">
    **Fetch data from your database in your API:**

    Update your service code to query the database. Example in Go:

    ```go
    import (
        "database/sql"
        "encoding/json"
        _ "github.com/lib/pq"
    )
    
    // Connect to database
    dbURL := os.Getenv("DATABASE_URL")
    db, err := sql.Open("postgres", dbURL)
    
    // Query users
    router.HandleFunc("GET /users", func(w http.ResponseWriter, r *http.Request) {
        rows, err := db.Query("SELECT id, email, created_at FROM users")
        // ... handle results
    })
    ```

    Generate Nitric client and test:

    ```bash
    make generate
    make run
    curl http://localhost:3000/users
    ```

    You'll see your user data:

    ```json
    [
      {"id":1,"email":"admin@example.com","created_at":"2025-08-14T17:12:05Z"},
      {"id":2,"email":"user@example.com","created_at":"2025-08-14T17:12:05Z"}
    ]
    ```
  </Accordion>
</AccordionGroup>

## Production Deployment

This setup works seamlessly with any PostgreSQL provider when deploying:

- **Neon**: Replace `DATABASE_URL` with your Neon connection string
- **Supabase**: Replace `DATABASE_URL` with your Supabase connection string
- **AWS RDS**: Replace `DATABASE_URL` with your RDS connection string

<Note>
  Stop with `make db-stop` when done.
</Note>
