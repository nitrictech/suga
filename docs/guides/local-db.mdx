---
title: 'Local Database Setup'
description: 'Set up PostgreSQL for Nitric local development.'
---

Nitric doesn't automatically provision databases in local development. Here's how to set up PostgreSQL for local development.

<AccordionGroup>
  <Accordion title="Step 1: Create Docker Compose" icon="docker">
    **Define your database container:**

    Docker Compose lets you run PostgreSQL without installing it locally. Create `docker-compose.yml` in your project root:

    ```yaml
    version: '3.8'

    services:
      postgres:
        image: postgres:15
        environment:
          POSTGRES_USER: admin
          POSTGRES_PASSWORD: secure_password
          POSTGRES_DB: myapp
        ports:
          - '5432:5432'
        volumes:
          - postgres_data:/var/lib/postgresql/data
          - ./init.sql:/init.sql

    volumes:
      postgres_data:
    ```
  </Accordion>

  <Accordion title="Step 2: Create Makefile" icon="file-code">
    **Add convenient database commands:**

    A Makefile simplifies managing your database. Create `Makefile`:

    ```makefile
    .PHONY: db-start db-stop db-reset

    db-start:
    	@echo "🐳 Starting database..."
    	@docker compose up -d
    	@echo "✅ Database ready!"

    db-stop:
    	@echo "🛑 Stopping database..."
    	@docker compose down

    db-reset:
    	@echo "🔄 Resetting database..."
    	@docker compose down -v
    	@docker compose up -d
    	@sleep 2
    	@docker compose exec postgres psql -U admin -d myapp -f /init.sql
    	@echo "✅ Database reset with fresh data!"
    ```
  </Accordion>
  
  <Accordion title="Step 3: Configure Nitric" icon="gear">
    **Register your database with Nitric:**

    Add your database to `nitric.yaml`:

    ```yaml
    databases:
      mydb:
        access:
          api:
            - query
    ```

    This tells Nitric about your database and which services can access it.
  </Accordion>

  <Accordion title="Step 4: Start & Connect" icon="plug">
    **Run your database and connect Nitric:**

    Start the database container:

    ```bash
    make db-start
    ```

    Tell Nitric where to find your database:

    ```bash
    export DATABASE_URL="postgresql://admin:secure_password@localhost:5432/myapp"
    ```

    Launch Nitric in development mode:

    ```bash
    nitric dev
    ```
  </Accordion>

  <Accordion title="Step 5: Initialize Database" icon="table">
    **Set up tables and seed data:**

    Create `init.sql` with your schema and initial data:

    ```sql
    DROP TABLE IF EXISTS users;
    
    CREATE TABLE users (
      id SERIAL PRIMARY KEY,
      email VARCHAR(255) UNIQUE NOT NULL,
      created_at TIMESTAMP DEFAULT NOW()
    );

    INSERT INTO users (email) VALUES
      ('admin@example.com'),
      ('user@example.com');
    ```

    Apply your schema with database reset:

    ```bash
    make db-reset
    ```

    <Note>
      For production, use a proper migration tool like golang-migrate, Flyway, or your ORM's migration system.
    </Note>
  </Accordion>
</AccordionGroup>

<Note>
  Stop with `make db-stop` when done.
</Note>
